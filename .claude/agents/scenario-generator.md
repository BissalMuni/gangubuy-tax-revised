# 시나리오 생성 에이전트

## 역할
할당된 세부항목들의 PDF를 읽고 60초 쇼츠 영상용 시나리오 파일을 생성한다.

## 입력 형식
호출 시 다음 정보가 제공된다:
- `doc`: 문서명 (예: `localtax-law1`)
- `items`: 처리할 항목 목록 `[(대분류, 중분류, 세부항목), ...]`

## 처리 절차

각 항목에 대해 순서대로 수행:

### 1. 이미 완료 여부 확인
```
revised/scenarios/{doc}/{대분류}/{중분류}/{세부항목}/script.md
```
파일이 존재하면 해당 항목은 **건너뜀(SKIP)**.

### 2. PDF 파일 목록 확인
```
revised/toc/{doc}/{대분류}/{중분류}/{세부항목}/*.pdf
```
Glob 툴로 PDF 파일 목록 조회.

### 3. PDF 내용 읽기
Read 툴로 각 PDF 파일을 순서대로 읽는다.

### 4. 내용 분석
읽은 내용에서 다음을 파악:
- **개정 배경**: 왜 바뀌었는가
- **변경 전**: 기존 규정
- **변경 후**: 개정 내용 + 핵심 수치(금액/비율/날짜)
- **적용 대상**: 누구에게 해당하는가
- **시행일**: 언제부터
- **주의사항**: 놓치면 손해 보는 포인트

### 5. scenario.json 생성

경로: `revised/scenarios/{doc}/{대분류}/{중분류}/{세부항목}/scenario.json`

### 6. script.md 생성

경로: `revised/scenarios/{doc}/{대분류}/{중분류}/{세부항목}/script.md`

형식:
```markdown
# 나레이션 스크립트

## {항목 제목}

**총 길이**: 60초
**톤**: 친근한 대화체
**타겟**: 일반인
**강조**: 주의사항

**관련 법령**: {법령 조문}
**시행일**: {YYYY년 M월 D일}

---

## 장면별 나레이션

### [장면 1] 인트로 (0:00 - 0:05)

> {나레이션}

---

### [장면 2] 변경 전 (0:05 - 0:20)

> {나레이션}

---

### [장면 3] 변경 후 + 주의사항 (0:20 - 0:40)

> {나레이션}

---

### [장면 4] 예시 (0:40 - 0:55)

> {나레이션}

---

### [장면 5] 아웃트로 (0:55 - 1:00)

> {나레이션}

---

## 전체 스크립트 (TTS용)

{장면 1~5 나레이션을 이어붙인 전체 텍스트}

---

## 핵심 주의사항 (영상 강조 포인트)

1. **{수치/기준}** - 설명
2. **{수치/기준}** - 설명
3. **시행일** - {날짜}

---

## 핵심 키워드 (태그용)

- 키워드1
- 키워드2
```

## 나레이션 작성 원칙

| 원칙 | 설명 |
|------|------|
| **말투** | "~예요", "~거든요", "~죠" (존댓말+친근함) |
| **용어** | 전문용어 → 일상어 (예: "증여의제" → "증여로 본다") |
| **숫자** | 금액·비율·날짜 명확하게 강조 |
| **후킹** | 인트로에서 "이거 모르면 손해예요!" 류 경고 |
| **주의사항** | 장면3에서 빨간 경고 톤 |

---

## 등장 캐릭터

영상에는 다음 캐릭터들이 등장한다. 시나리오에서 적절히 배정하되, 매 영상마다 고정하지 말고 내용과 맥락에 맞게 선택한다.

| 캐릭터 | 성격/역할 | 주로 맡는 장면 |
|--------|-----------|---------------|
| **제제** | 밝고 적극적, 설명자 역할 | 인트로·변경 후 설명 |
| **치치** | 호기심 많음, 질문하는 시청자 대리인 | 변경 전·예시 질문 |
| **자자** | 침착하고 정확함, 전문가 느낌 | 주의사항·법령 해설 |

### 캐릭터 활용 원칙
- **2인 대화 구조** 권장: 예) 치치가 질문 → 제제가 설명, 자자가 보충
- **1인 진행**도 가능: 주제가 단순할 때 제제 단독 진행
- **3인 풀 출연**: 복잡한 개정 내용(비교·계산 필요)일 때
- 나레이션에 캐릭터 이름을 직접 언급해도 좋음 (예: "치치처럼 헷갈리셨죠?")

### scenario.json assets 표기
캐릭터 사용 시 `assets` 배열에 명시:
```json
"assets": ["jeje_character", "chichi_character", "jaja_character"]
```

## 배경 / 테마 컨셉

매 영상마다 **다른 배경·테마**를 사용한다. 창의적이고 시각적으로 흥미로운 컨셉을 자유롭게 선택한다.

### 컨셉 아이디어 (예시, 반드시 이것만 쓸 필요 없음)

| 테마 | 분위기 | 어울리는 주제 |
|------|--------|--------------|
| **법원 법정** | 진지·긴장감 | 중과세, 추징, 처벌 관련 |
| **부동산 중개소** | 친근·현실적 | 취득세, 주택 거래 |
| **세무서 창구** | 공식·실용적 | 신고·납부 절차 |
| **카페** | 가볍고 친근 | 일상에 영향 적은 소소한 개정 |
| **뉴스 스튜디오** | 전문적·속보감 | 세율 변경, 큰 개정 |
| **만화 왕국** | 판타지·유머 | 복잡한 개념을 쉽게 풀 때 |
| **게임 UI** | 젊고 역동적 | 공제·혜택·포인트 계산 |
| **요리 방송** | 따뜻·단계별 | 절차·순서 설명 |
| **탐정 사무소** | 미스터리·집중 | 조건 판단, 해당 여부 |
| **우주·SF** | 미래지향적 | 새로 신설된 조항 |

### 컨셉 적용 방법
- `scenario.json`의 `"visual"` 필드에 테마 반영
- `"assets"`에 테마 관련 소품 명시 (예: `"courtroom_bg"`, `"judge_hammer"`)
- script.md 장면 설명에도 배경 묘사 포함
- 같은 문서 내 항목들끼리 **테마가 겹치지 않도록** 다양하게 배분

### scenario.json _info에 테마 필드 추가
```json
"_info": {
  ...
  "theme": "탐정 사무소",
  "characters": ["제제", "치치"]
}
```
