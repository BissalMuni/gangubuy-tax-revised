# 시나리오 생성 워크플로우

## 개요
목차별로 분류된 PDF를 읽어 개정세법 설명 쇼츠 영상용 시나리오를 생성하는 워크플로우

## 입력
```
revised/toc/[문서명]/[대분류]/[중분류]/[세부항목]/
├── p009.pdf
├── p010.pdf
└── ...
```

## 출력
```
scenarios/[문서명]/[대분류]/[중분류]/[세부항목]/
├── scenario.json       # 장면별 시나리오
├── script.md           # 나레이션 스크립트
└── storyboard.md       # 스토리보드 (선택)
```

## 시나리오 구조

    "duration_target": "60초"

  "scenes": [

      "text_overlay": "2026 지방세법 개정"

## 실행 단계

### 1단계: 세부항목 PDF 읽기
```
# Claude Code에서 특정 세부항목 폴더의 PDF 읽기
revised/toc/localtax-law1/Ⅰ_취득세/지방세법/01_배우자_직계존비속_증여의제/
```

### 2단계: 내용 분석
- 개정 전후 비교 내용 파악
- 핵심 변경사항 추출
- 적용 대상 및 시기 확인

### 3단계: 시나리오 생성
- 60초 쇼츠 기준으로 장면 구성
- 각 장면별 나레이션 작성
- 시각적 요소 설명

### 4단계: 스크립트 작성
`script.md` 파일에 나레이션 전문 작성

## 시나리오 생성 가이드라인

### 콘텐츠 기조

| 항목 | 설정 |
|------|------|
| **톤/말투** | 친근한 대화체 (반말X, 존댓말+친근함) |
| **타겟 시청자** | 일반인 (세금 비전문가) |
| **강조 포인트** | 주의사항 (놓치면 손해보는 것) |
| **영상 길이** | 60초 |

### 나레이션 작성 원칙

1. **친근한 대화체**
   - "~입니다" 보다 "~예요", "~거든요", "~죠"
   - 시청자에게 말 걸듯이: "혹시 ~하시는 분들 계신가요?"
   - 공감 표현: "많이들 헷갈리시죠?", "이거 모르면 손해예요!"

2. **쉬운 용어**
   - 전문 용어 → 일상 용어로 변환
   - 예: "증여의제" → "증여로 본다", "시가인정액" → "시세"
   - 법조문 인용 최소화

3. **주의사항 강조**
   - "꼭 기억하세요!", "이것만은 주의!"
   - 손해 보는 상황 구체적으로 제시
   - 경고 톤 적절히 활용

4. **숫자는 명확하게**
   - 금액, 비율, 날짜 강조
   - 예: "3억원", "30%", "2026년 1월 1일"

### 쇼츠 영상 구조 (60초 기준)

| 장면 | 시간 | 유형 | 목적 |
|------|------|------|------|
| 1 | 5초 | intro | 주제 소개 + 관심 유발 |
| 2 | 15초 | before | 기존 규정 (쉽게 설명) |
| 3 | 20초 | after | 개정 내용 + **주의사항** |
| 4 | 15초 | example | 구체적 사례 (숫자로) |
| 5 | 5초 | outro | 마무리 + CTA |

### 등장 캐릭터

영상에는 다음 캐릭터들이 등장한다. 매 영상마다 고정하지 말고 내용과 맥락에 맞게 선택한다.

| 캐릭터 | 성격/역할 | 주로 맡는 장면 |
|--------|-----------|---------------|
| **제제** | 밝고 적극적, 설명자 역할 | 인트로·변경 후 설명 |
| **치치** | 호기심 많음, 질문하는 시청자 대리인 | 변경 전·예시 질문 |
| **자자** | 침착하고 정확함, 전문가 느낌 | 주의사항·법령 해설 |

**활용 원칙**
- **2인 대화 구조** 권장: 치치가 질문 → 제제가 설명, 자자가 보충
- **1인 진행**: 주제가 단순할 때 제제 단독 진행
- **3인 풀 출연**: 복잡한 개정 내용(비교·계산 필요)일 때
- 나레이션에 캐릭터 이름 직접 언급 가능 (예: "치치처럼 헷갈리셨죠?")

### 배경 / 테마 컨셉

매 영상마다 **다른 배경·테마**를 사용한다. 창의적이고 시각적으로 흥미로운 컨셉을 자유롭게 선택한다.

| 테마 | 분위기 | 어울리는 주제 |
|------|--------|--------------|
| **법원 법정** | 진지·긴장감 | 중과세, 추징, 처벌 관련 |
| **부동산 중개소** | 친근·현실적 | 취득세, 주택 거래 |
| **세무서 창구** | 공식·실용적 | 신고·납부 절차 |
| **카페** | 가볍고 친근 | 일상에 영향 적은 소소한 개정 |
| **뉴스 스튜디오** | 전문적·속보감 | 세율 변경, 큰 개정 |
| **만화 왕국** | 판타지·유머 | 복잡한 개념을 쉽게 풀 때 |
| **게임 UI** | 젊고 역동적 | 공제·혜택·포인트 계산 |
| **요리 방송** | 따뜻·단계별 | 절차·순서 설명 |
| **탐정 사무소** | 미스터리·집중 | 조건 판단, 해당 여부 |
| **우주·SF** | 미래지향적 | 새로 신설된 조항 |

- 같은 문서 내 항목들끼리 **테마가 겹치지 않도록** 다양하게 배분
- `scenario.json`의 `_info`에 `"theme"`, `"characters"` 필드 추가

### 시각 요소 가이드
- 비교 장면: 좌우 분할 또는 전후 전환
- 숫자/금액: 큰 텍스트 오버레이
- 캐릭터 애니메이션: 제제·치치·자자 활용
- **주의사항**: 빨간색/노란색 강조, 경고 아이콘

## 작업 체크리스트

1. [ ] 세부항목 폴더 선택
2. [ ] PDF 페이지 모두 읽기
3. [ ] 개정 전후 내용 분석
4. [ ] scenario.json 생성
5. [ ] script.md 작성
6. [ ] 검토 및 수정

## 예시: 시나리오 생성 프롬프트

```
다음 PDF 내용을 읽고 60초 쇼츠 영상용 시나리오를 생성해줘:

[PDF 내용]

요구사항:
1. 5개 장면으로 구성 (인트로, 변경전, 변경후, 예시, 아웃트로)
2. 각 장면별 나레이션과 시각 요소 포함
3. 일반인이 이해하기 쉬운 언어로 작성
4. scenario.json 형식으로 출력
```
